HALLAZGO CRÍTICO #1 — Discrepancia de nombre de vista: Dim_Produtos vs Dim_Productos
Módulo afectado: get-inventario-data.ts y get-inventario-filters.ts

Problema: El CSV registra el view como Dim_Productos (español, con 'c'). El código usa dbo.Dim_Produtos (portugués, con 't'). Si el nombre real en Azure SQL es Dim_Productos, todos los queries de Inventario fallarían con Invalid object name.


-- CSV dice:
Dim_Productos ; SK_Produto

-- Código usa:
INNER JOIN dbo.Dim_Produtos dp ON fi.id_produto = dp.SK_Produto
Acción requerida: Confirmar el nombre exacto del objeto en Azure SQL y corregir el código o el CSV. Verificar antes del despliegue.

HALLAZGO MEDIO #12 — Usuario sin sucursal asignada ve TODOS los datos
Módulo afectado: sucursalFilter() en todos los módulos


@isSupervisor = 0 AND (@sucursalId IS NULL OR col = @sucursalId)
Si @sucursalId IS NULL y el usuario no es supervisor, la condición retorna TODAS las sucursales. Un usuario de sucursal A que no seleccione filtro obtiene datos globales. Validar si esta es la regla de negocio intencional para el rol ADMINISTRADOR.

HALLAZGO BAJO #13 — Middleware no redirige /dashboard raíz para supervisores
Módulo afectado: middleware.ts:12

El middleware protege /dashboard/usuarios y /dashboard/sucursales para supervisores, pero si un supervisor accede directamente a /dashboard (raíz), pasa sin redirección. El page.tsx de dashboard debería manejar esto, pero no está explícito en el middleware.

HALLAZGO BAJO #14 — console.error(err) puede exponer stack traces en logs
Todos los Server Actions hacen console.error("[Módulo]", err). Esto es correcto para servidor, pero en producción Azure App Service estos logs son accesibles. Se recomienda separar el log técnico del mensaje enviado al cliente (ya implementado), pero agregar niveles de severidad.
